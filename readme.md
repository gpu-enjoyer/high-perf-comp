```bash
g++ --version
```

## Запуск

- Измените параметры запуска в файлах `config/`
- Запустите `run.sh`
- В случае успешного выполнения графики отобразятся в `results/`

Исходный код каждой задачи находится в `code/`


## Формат исследования

### Параметры экспериментов

- Число потоков относительно числа ядер (меньше, столько же, больше)
- Масштаб задачи (размеры массивов или матриц)

### Измерения

- time
- speedup


## Условия задач

[tasks.pdf](tasks.pdf)

1. **Минимальный (максимальный) элемент вектора.**  
Рассмотреть вариант с использованием редукции и без.

2. **Скалярное произведение двух векторов.**

3. **Определенный интеграл методом прямоугольников.**

4. **Максимальное значение среди минимальных элементов строк матрицы.**  
Такая задача имеет место для решения матричных игр.

5. **Предыдущая задача, используя матрицы специального типа (ленточные, треугольные и другие).**  
Рассмотреть **различные правила распределения итераций между потоками**, сравнить эффективность параллельных вычислений.  
Выполнение таких экспериментов целесообразно для задач, в которых вычислительная трудоемкость итераций циклов различна.

6. **Исследовать режимы распараллеливания цикла for: static, dynamic, guided.**  
**Придумать цикл с неравномерной нагрузкой итераций** (например, на некоторых итерациях генерировать случайные числа).  
Сравнить, как будет изменяться время работы такой программы при различных режимах работы цикла for.

7. **Реализовать операцию редукции с использованием различных способов организации взаимоисключения (атомарные операции, критические секции, синхронизация при помощи замков).**  
Оценить эффективность разных подходов.  
**Сравнить с быстродействием операции редукции, выполняемой посредством параметра reduction директивы for.**

8. **Скалярное произведение последовательного набора векторов.**  
Исходные данные можно подготовить заранее в отдельном файле.  
**Организовать ввод векторов и вычисление произведения как две параллельные задачи, используя директиву sections.**

9. **Разработать программу с использованием и без использования вложенного параллелизма** (если компилятор поддерживает вложенные параллельные фрагменты).  
Достаточно разработать программу для одной из задач (например, задачи 4) с использованием распараллеливания циклов разного уровня вложенности.  
Выполнить вычислительные эксперименты и оценить эффективность разных подходов.
